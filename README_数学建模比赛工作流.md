# 数学建模比赛工作流 README

这份 README 记录本次华中杯 A 题从模板、建模、算法、图表、排版到提交检查中踩过的坑和沉淀下来的做法。下次参加数模比赛时，可以把它当作赛前和赛中的作战手册。

## 0. 总原则

数模论文不是作品报告，也不是算法堆料展示。高分论文要同时做到四件事：

1. 题意口径准：题面明确约束优先于个人几何假设或直觉解释。
2. 模型逻辑闭环：每个变量、约束、目标函数、参数来源都能解释。
3. 算法有针对性：算法服务于题目结构，而不是为了显得高级而堆名词。
4. 图表承担论证：图不是装饰，必须直接回答“为什么这个模型有效”。

最重要的一句话：正文讲逻辑和结果，附录放证据和细节。

## 1. 赛前准备

建议提前准备一套可复用 LaTeX 工程，至少包含：

- 摘要页、问题重述、模型假设、符号说明、数据处理、模型建立、算法设计、结果分析、敏感性分析、模型评价、参考文献、附录。
- 中文正文宋体，英文和数字 Times New Roman。
- 自动编号的公式、图、表和交叉引用。
- 代码高亮环境，附录中代码自动换行，避免越界。

## 2. 开赛后先做什么

拿到题后不要马上写论文，也不要马上上复杂算法。先做四步：

1. 审题：列出每问输入、输出、硬约束、软约束、评价指标。
2. 查附件：确认数据字段、缺失值、异常值、单位、时间口径、坐标口径。
3. 做基线：先用最朴素可行方法跑出第一版结果，保证题目可计算。
4. 定主线：明确每问之间的递进关系，论文围绕一条技术路线展开。

本次最大教训：绿色配送区“30个客户”和“10 km 半径”冲突时，应优先服从题面文字约束，把 30 客户口径作为主模型，10 km 口径放入敏感性分析。

## 3. 数据处理写法

数据处理不要只写“清洗后得到多少条”。要说明为什么这样处理。

推荐结构：

- 缺失值：优先用有物理意义的规则，例如物流中用密度 `重量/体积` 推断缺失量，而不是无脑均值。
- 异常值：说明异常判别规则，例如超容量、负值、时间窗逆序。
- 任务拆分：不能过早把大订单全部切碎。若必须拆分，要说明后续算法中有 Split/Merge 算子恢复大车协同优势。
- 装箱规则：说明虚拟任务只是计算粒度，不是业务上强制拆单。

正文放一张数据流水图和一个清洗统计表即可，长数据表放附录或结果文件。

## 4. 模型建立写法

模型部分要“像数学论文”，不能像项目说明。

必要元素：

- 集合、变量、参数、决策变量。
- 目标函数各项成本口径统一，例如固定成本、能源成本、碳排放成本、时间窗惩罚都转化为元。
- 约束逐组说明：路径流守恒、车辆容量、时间递推、软时间窗、车型可服务性、政策限制。
- 对硬编码常数使用符号，例如 `T_lim=960` 表示 16:00，不要在公式里裸写 960。
- 服务时间应由固定停车时间和变动卸货时间组成：

```tex
\delta_i = \delta_0 I_i + \alpha w_i
```

时变路网中若速度阶跃变化，要考虑 FIFO 悖论。可用节点等待机制：

```tex
t_j = t_i + \delta_i + w_i + \tau_{ij}(t_i+\delta_i+w_i)
```

并解释等待在现实中对应延迟发车、司机休息、补电或调度缓冲。

## 5. 算法选择

算法不是越多越好。选择算法时看题目结构。

VRP 类问题推荐路线：

- 小规模：MIP 或枚举/动态规划求精确解，用于验证 Gap。
- 中大规模：构造初解 + ALNS + SA 接受准则。
- 有禁行政策：加入政策相关破坏算子。
- 有动态事件：滚动时域 RH，只重排未执行部分。
- 有不确定订单：预测模型只做先验或虚拟需求，不替代确定性优化。

不要做的事：

- 明明能精确计算的成本，硬上 DNN 分类器预测。
- SA、TS、GA、DRL 全塞进去，却说不清每个组件解决什么问题。
- 用强算法只和最弱贪心比，缺少 GA、TS 或小规模精确解对比。

推荐验证组合：

- 与 SIH、GA、TS 做横向收敛对比。
- 做 ALNS 消融实验：无 SA、单起点、无前瞻性插入。
- 抽 10 到 15 个任务做精确验证，报告 Optimality Gap。
- 做参数正交试验或网格搜索，别让参数像拍脑袋。

## 6. AI 或机器学习模块怎么写

AI 模块必须解决真实不确定性，不能为了热点硬加。

更合理的落点：

- 预测未来路况速度。
- 预测新增订单热区概率。
- 估计动态事件发生概率。
- 把预测概率转为两阶段随机规划 TSSP 的虚拟需求。
- 在 ALNS 插入算子中加入前瞻性容量预留。

如果数据只有单日小样本，必须降调表述：

- 不说“可上线泛化预测模型”。
- 说“基于封闭数据的截面规律提取器”或“热区先验提取器”。
- 报告交叉验证，并承认真实上线需要长期历史流水。

常用术语第一次出现要写全称：

- 随机森林（Random Forest, RF）
- 滚动时域（Rolling Horizon, RH）
- 两阶段随机规划（Two-Stage Stochastic Programming, TSSP）
- 自适应大邻域搜索（Adaptive Large Neighborhood Search, ALNS）
- 受试者工作特征曲线（Receiver Operating Characteristic, ROC）
- 曲线下面积（Area Under Curve, AUC）

## 7. 图表设计原则

图表要像论文，不要像 AI 生成的流程海报。

通用原则：

- 图内不要放大标题，标题交给 LaTeX caption。
- 少用大段文字和过多方框，必要节点保留英文短词或中英混排。
- 不要让数值标签和柱子、坐标轴、图例重叠。
- 图中不要全英文，也不要全中文堆满。术语、层名、输入输出可用短英文。
- 配色克制，避免紫蓝渐变、AI 感强的高饱和配色。
- 尽量导出 PDF 矢量图，正文中插 PDF，清晰度优先。

VRP 论文常用高价值图：

- 数据处理流水图。
- 客户空间分布图。
- 重量-体积散点图。
- 时间窗热力图。
- 成本构成图。
- 路径距离与成本散点/箱线图。
- 路线图。
- 绿色限行时间线。
- 车辆甘特图。
- 时空轨迹图。
- 算法收敛曲线。
- 消融实验曲线。
- 参数调优响应面。
- 精确 Gap 验证图。
- 动态事件流压力测试图。
- Pareto 前沿图。
- 敏感性热力图。
- RF 混淆矩阵、ROC 曲线、特征重要性图。

每张图必须在正文中被明确引用，例如：

```tex
整体建模与求解流程见图\ref{fig:model-flow}。
```

不要出现只有图，没有解释的“孤儿图”。

## 8. 摘要写法

摘要必须一页内完成，且要有硬数字。

推荐结构：

1. 一句话说明问题类型和总体方法。
2. 每问一段，写模型、算法、核心结果。
3. 写相对提升率，而不只是绝对值。
4. 最后一段写管理启示。

摘要应包含：

- 启用车辆数。
- 总成本。
- 总距离。
- 碳排放。
- 政策导致的成本增量。
- 动态调度响应时间。
- 算法相对基线提升。

摘要不要写成“本文建立了模型并得到了较好结果”。评委没有时间猜。

## 9. 敏感性分析怎么做

敏感性分析不是凑图，而是回答“政策或参数变了，结论稳不稳”。

推荐方向：

- 碳价变化：总成本、排放量、新能源车比例的变化。
- 迟到罚系数变化：服务质量和物理成本的 Pareto 权衡。
- 绿色区半径或客户数口径变化：政策范围冲击。
- 限行时长变化：找到成本非线性增长临界点。
- 拆分阈值变化：任务数量、求解速度、总成本之间的关系。

重点是给出洞察，例如：

- “限行延长到 20:00 后成本增幅明显扩大。”
- “碳价上升会推动新能源车承担更多绿色区任务。”
- “过大的拆分阈值减少任务数，但会削弱组合灵活性。”

## 10. 参考文献与资料

参考文献必须真实存在，优先用：

- 官方政策或统计数据。
- 经典英文论文或教材。
- 中文核心/正式期刊论文。
- 与模型直接相关的算法文献。

注意事项：

- 不要把赛题附件或组委会说明当作参考文献放入文末。
- 引用编号应按正文首次出现顺序排列。
- 文献都要在正文中被引用。
- 中文论文最好保留网页核验材料或 DOI。
- 政策数据要保存网页或 PDF 佐证。

## 11. 附录与佐证材料

正文控制信息密度，附录放证据。

适合放附录的内容：

- 支撑文件清单。
- 算法伪代码。
- 关键代码片段。
- 参数调优补充图。
- 精确验证补充图。
- 长表格和完整路径结果。

佐证材料建议分六类：

```text
01_论文正文
02_求解代码
03_仿真
04_数据与结果
05_论文图表
06_参考文献与政策数据
```

提交前要同步正文、代码、结果、图表，避免论文是新版、佐证材料是旧版。

## 12. 提交前最终检查

匿名性检查：

- PDF 元数据不能有作者、学校、用户名。
- 正文、页眉页脚、截图、文件名不能出现姓名、学校、指导老师。
- LaTeX 中设置 `pdfauthor={}`。

图表检查：

- 所有图表在正文中都有引用。
- 放大 150% 到 200% 检查文字是否模糊。
- 检查标签是否遮挡柱子、坐标轴、图例。
- 图内不要出现大标题，标题由 caption 管。

LaTeX 检查：

```bash
xelatex -interaction=nonstopmode main.tex
xelatex -interaction=nonstopmode main.tex
rg -n "Overfull|Undefined control sequence|Citation .* undefined|Reference .* undefined|Label\\(s\\) may have changed" main.log
```

PDF 文本和元数据检查：

```bash
python3 - <<'PY'
from pdfminer.high_level import extract_text
from pdfminer.pdfparser import PDFParser
from pdfminer.pdfdocument import PDFDocument
from pathlib import Path

p = Path("main.pdf")
with p.open("rb") as f:
    doc = PDFDocument(PDFParser(f))
    print(doc.info)

text = extract_text(str(p))
terms = ["姓名", "学校", "学院", "指导老师", "指导教师", "作者", "/Users", "/Volumes"]
print([t for t in terms if t in text])
PY
```

图表孤岛检查思路：

- 搜索所有 `\label{fig:...}` 和 `\label{tab:...}`。
- 搜索所有 `\ref{...}`。
- 确保每个图表 label 至少被引用一次。

参考文献检查：

- `.aux` 文件中的首次 `\citation{}` 顺序应与 `\bibcite{}` 编号一致。
- 文献总数和引用数尽量一致，避免文末有没被引用的文献。

提交包检查：

```bash
zip -r -X submit.zip 提交目录 -x "*/._*" "*/.DS_Store" "*/~$*"
unzip -l submit.zip
```

尤其注意排除：

- `._*`
- `.DS_Store`
- Word 临时文件 `~$*`
- 无关草稿和旧版 PDF

## 13. 本次华中杯 A 题的关键经验

这次最值得记住的点：

- 不要强行深度学习。AI 模块应服务于不确定性，例如新增订单热区，而不是预测一个本可由目标函数精确计算的结果。
- 题面口径优先。30 个绿色区客户应作为主模型，10 km 半径作为敏感性分析。
- 晚高峰不能忽略。17:00 任务峰值与真实拥堵一致，应主动补充 17:00--19:00 拥堵假设。
- 异构车队要体现异构性。车型固定成本、容量、能耗不能一刀切。
- 拆分订单要防止过度碎片化。静态拆分后最好配合动态 Split/Merge。
- 动态调度不能只看成本，还要看扰动度，例如 RDI。
- 好图比长文字有力。甘特图、时空轨迹图、Pareto 图、收敛曲线，是数模论文里的“高密度证据”。
- 最后半天不要再大改模型，重点做匿名性、引用、图表、页码、提交包检查。

## 14. 下次开赛的最短行动清单

1. 建 LaTeX 工程，确认能编译。
2. 读题，写三问输入输出表。
3. 跑数据概览，生成缺失值、异常值、空间分布、时间分布图。
4. 写一个朴素基线，先得到可行解。
5. 建主模型，公式和符号表同步。
6. 设计针对性算法，不乱堆。
7. 生成论文图，优先 PDF 矢量。
8. 写正文，图表跟着论证走。
9. 做基线对比、消融、敏感性、精确小算例验证。
10. 最后统一数值、引用、匿名性和提交包。

记住：数模比赛最后拼的不是“看起来用了多少高级词”，而是有没有把题意、模型、算法、数据和论文表达拧成一股绳。
